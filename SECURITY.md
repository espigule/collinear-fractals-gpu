# Security Policy

## Supported versions

Only the latest tagged release receives security and integrity updates.

## Reporting a vulnerability

For security-sensitive reports, contact the maintainer privately before opening a public issue. If no private contact channel is listed on the GitHub profile, open a minimal public issue that requests a private disclosure channel and does not include exploit details.

For numerical correctness, reproducibility, or certificate-integrity concerns that are not security-sensitive, use the research validation issue template.

## Scope

This is static research software with no server-side runtime. Security-sensitive areas are primarily release integrity, supply-chain metadata, generated artifacts, and browser-side file handling.
